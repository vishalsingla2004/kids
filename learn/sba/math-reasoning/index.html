1. Always / Sometimes / Never
Classification Rule
Test with examples AND counterexamples
To classify a statement, test it with many examples. If it's true for all cases: "always." If true for some but not all: "sometimes." If never true: "never." One counterexample disproves "always."
Worked Example
"A square is a rectangle" → Always (all squares have 4 right angles)
"A rectangle is a square" → Sometimes (only when sides are equal)
2. True or False Equations
Verification Method
Evaluate both sides separately
To verify an equation, calculate each side independently. If both sides equal the same number, the equation is true. Don't assume — always show your work for each side.
Worked Example
Is 4 × (3 + 2) = 4×3 + 4×2?
LHS = 4 × 5 = 20. RHS = 12 + 8 = 20.
TRUE ✓ (Distributive Property)
3. Showing Your Work
Work Format
Write each step on its own line
Clear work means every step is visible and labeled. Write the problem, then show each calculation on a new line, and end with a clearly labeled answer. This makes it easy to find any errors.
Worked Example
Problem → Step 1: find LCD → Step 2: convert → Step 3: add → Answer with label
4. Finding Errors
Error-Finding Strategy
Check each step against the rules
When reviewing work, check every step. Look for skipped steps, wrong operations, or arithmetic mistakes. Explain exactly what rule was violated and how to fix it.
Worked Example
"5/6 + 1/3 = 6/9 = 2/3" — Error: added numerators AND denominators directly.
Fix: LCD = 6, so 1/3 = 2/6, then 5/6 + 2/6 = 7/6 = 1⅙
5. Using Counterexamples
Counterexample Rule
One counterexample disproves "always"
A counterexample is a single case where a claim fails. You only need to find one to show a statement is false. Counterexamples must be valid examples that clearly break the rule.
Worked Example
"All rectangles are squares" → counterexample: a 4×2 rectangle has unequal sides → claim is FALSE
6. Multiple Representations
Representations
words ↔ picture ↔ number ↔ equation
The same mathematical idea can be shown in many ways. Being able to switch between representations shows deep understanding. All forms of 3/4 represent the same amount.
Worked Example
3/4 = 0.75 = 75% = "three-fourths" = shaded bar (3 of 4 parts) — all the same value
🟡 Medium
7. Constructing Arguments
Argument Structure
Claim → Evidence → Reasoning
A complete mathematical argument has three parts: a clear claim (what you believe), evidence (numbers or examples proving it), and reasoning (why the evidence supports the claim).
Worked Example
Claim: "3/8 < 1/2"
Evidence: 3/8 = 3/8; 1/2 = 4/8
Reasoning: 3 < 4, so 3/8 < 4/8 = 1/2 ✓
8. Justifying Fraction Ordering
Fraction Comparison
Use common denominators or benchmarks
To compare fractions, convert both to the same denominator, then compare numerators. You can also use benchmark fractions (0, 1/2, 1) to estimate relative size.
Worked Example
Is 1/3 < 3/8?
LCD = 24: 1/3 = 8/24, 3/8 = 9/24
8/24 < 9/24, so 1/3 < 3/8 ✓
9. Generalizing Patterns
Generalization Method
Test rule on multiple examples
After noticing a pattern, write it as a general rule. Test the rule with many different examples. The more examples that work, the more confident you can be in the rule.
Worked Example
2+4=6, 3+5=8, 10+12=22 → all even
Rule: "even + even = even" ✓
10. Comparing Methods
Comparison Rule
Both can be correct; explain trade-offs
Different methods can lead to the same correct answer. When comparing, explain what makes each efficient or inefficient. Neither is wrong; one may be more practical in certain situations.
Worked Example
Finding LCD vs multiplying denominators: both give the right answer. LCM gives smaller numbers; multiplying works but may need simplifying at the end.
11. Logical Chains
Logical Chain
If A → B → C, then A → C
In logical chains, each step follows from the previous one. If you can show A leads to B and B leads to C, you've proven A leads to C. This is how shape hierarchies work.
Worked Example
"If a shape has 4 equal sides AND 4 right angles, THEN it's a square."
Square → Rectangle → Parallelogram: so squares are always parallelograms
12. Explaining Volume
Volume Explained
V = B × h (B = base area)
Each layer = l×w cubes; h layers total
The base area (B = l × w) counts the cubes in one layer. Multiplying by height (h) stacks those layers. This is WHY V = l × w × h — it's not just a formula, it describes physical stacking.
Worked Example
Box 4 × 3 × 2: base has 4×3 = 12 cubes per layer. 2 layers → 12 × 2 = 24 cubic units ✓

13. Complete Proofs
Complete Proof Format
Claim → Evidence → General Conclusion
A full proof makes a claim, proves it with at least 2–3 numerical examples, and then states a general conclusion. The conclusion should explain why it always works, not just that it worked for these examples.
Worked Example
Claim: "×1/2 = ÷2"
Evidence: 8×(1/2)=4 and 8÷2=4; 10×(1/2)=5 and 10÷2=5
Conclusion: always equal for any whole number
14. Critique Arguments
Critique Framework
Find what's right, wrong, and missing
When critiquing someone's math, find what they did correctly, identify the specific error, and explain what rule or step is missing. A good critique is constructive and precise.
Worked Example
"0.5 > 0.49 because 5 > 49" — right answer, wrong reasoning.
Fix: Compare as 0.50 > 0.49 (same decimal places). The reasoning needs to match decimal place values.
15. Abstract Justification
Algebraic Justification
Use variables to show patterns hold for all numbers
Using variables (letters) instead of specific numbers allows you to show a rule works for ALL values, not just one example. The area model proves the distributive property for any rectangle dimensions.
Worked Example
Why does (a+b)×c = a×c + b×c?
Area model: total area = left part + right part
Works for ANY values of a, b, and c
16. Multiple Strategies
Multiple Paths
Different paths to same answer confirm correctness
Solving the same problem two different ways and getting the same answer confirms your answer is correct. Show both methods clearly and identify where they connect.
Worked Example
3/4 ÷ 1/2:
Draw: 3/4 has 1½ halves (visual)
Formula: 3/4 × 2/1 = 6/4 = 1½ ✓ (both methods agree)
17. Edge Cases
Testing Edge Cases
Test 0, extremes, and values > 1
Before declaring a rule always true, test edge cases: 0, 1, fractions greater than 1, and very large or small numbers. Many rules only hold within certain constraints.
Worked Example
"Product of two fractions is always smaller" →
1/2 × 1/2 = 1/4 ✓ smaller
BUT 3/2 × 3/2 = 9/4 > 1 ✗
Only true for fractions < 1
18. Polygon Hierarchy Proof
Hierarchy Rule
More specific shape inherits all parent properties
Each level inherits all properties of the level above. A square is inside the Rectangle set, which is inside Parallelogram, etc. This means squares have ALL properties of all outer categories.
Worked Example
A rhombus has 4 equal sides, variable angles.
A square has 4 equal sides AND 4 right angles.
Squares are always rhombuses, but rhombuses aren't always squares.
